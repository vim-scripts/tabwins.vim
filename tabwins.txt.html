 35 Section 1: Introduction                         TabwinsIntroduction
 36 ====================================================================
 37 
 38 See the screen shots at
 39   https://github.com/cesheridan/graphics/tree/master/tabwins
 40 
 41 
 42 tabwins.vim creates and enables definition of Vim commands that
 43 BUILD TABS with:
 44 
 45   1) SYMMETRIC  window structures of EMPTY BUFFERS 
 46 
 47      e.g. for 4 columns, each with 2 windows --
 48        :TabwinsVertical 2222  OR
 49        :Tabwins         2222  OR
 50        :V4x2
 51 
 52 
 53      e.g. for 2 columns, each with 12 windows --
 54         :TabwinsVertical [ 12, 12 ]  OR
 55         :Tabwins         [ 12, 12 ]  OR
 56         :V2x12
 57 
 58 
 59 
 60   2) ASYMMETRIC window structures of EMPTY BUFFERS
 61 
 62      e.g. for 4 columns, of 2, 1, 1, & 3 windows successively --
 63        :TabwinsVertical 2113
 64        :Tabwins         2113
 65 
 66      e.g. for 4 columns, of 18, 5, 26, & 3 windows successively --
 67        :TabwinsVertical  [ 18, 5, 26, 3 ]
 68        :Tabwins          [ 18, 5, 26, 3 ]
 69 
 70        => Use a list arg for dimensions >9, as above.
 71 
 72 
 73   3) EITHER of the above structures, with BUFFERS POPULATED
 74      by files, dirs, or results of running :execute on
 75      developer-specified strings.
 76 
 77      e.g. :Open_tab_unix_filesystem_1 for an asymmetric V234 tab
 78      populated with typical unix dirs.
 79 
 80 Loading tabwins.vim creates Vim commands that have name formats
 81 as described in section TabwinsUsage of tabwins.txt.
 82 
 83 See section TabwinsVandH for definitions of Vertical and Horizontal
 84 tabs, and of Primary and Secondary axes.
 85 
 86 Default settings load Vim menu Tabwins, for access to selected
 87                       ~~~~~~~~~~~~~~~~
 88 tab-builder commands.  Those and additional tab builders are
 89 available via Vim commands defined in tabwins.vim. The menu has
 90 only a small portion of the tab builders defined in tabwins.vim,
 91 and the developer is encouraged to add tab builder commands as needed.
 92 
 93 By enabling customized tab creation ~during~ Vim sessions,
 94 tabwins.vim gives each developer immediate access to the Vim
 95 window structures and content that the developer uses the most.
 96 
 97 The developer
 98   avoids having to reload backed up Vim sessions,
 99   avoids losing the current configuration, and
100   avoids loss of focus caused by these discontinuities.
101 
102 Easy access to the power of the Vim GUI.
103 
104 
105 
106 This plugin is structured for deployment in a pathogen-managed
107 bundle directory.
108 
109 See the rest of tabwins.txt for more information.

112 ====================================================================
113 Section 2: Usage                                       TabwinsUsage
114 ====================================================================
115 
116   Loading tabwins.vim creates Vim commands that have name formats
117   as described below.
118 
119   See TabwinsVandH for background about Vertical/Horizontal
120   and primary/secondary axes.
121 
122 
123 
124   DEFAULT AXIS :Tabwins 
125 
126   :Tabwins is an alias for :TabwinsVertical, meaning
127   that the Vertical axis is the default primary axis.
128 
129 
130 
131   WINDOWS OF EMPTY BUFFERS, SYMMETRIC & ASYMMETRIC
132 
133   1) :V{N},  :H{N}, :TabwinsVertical {N}, :TabwinsVertical { [ I ] }
134      e.g. :V3, :H4, :TabwinsVertical   4, :TabwinsVertical [ 12 ]
135                     :Tabwins           4, :Tabwins [ 12 ]
136 
137     One-dimensional tabs, w/ primary axis identifed by V or H, and
138     w/ N windows.
139 
140     E.g. :V3 creates a tab of 3 vertical windows, and :H4 creates
141     a tab of 4 horizontal windows.
142 
143     Loading tabwins.vim creates :HN & :VN commands where N is from
144     1 to the value of the relevant axis size in:
145        g:tabwins_max_v_size
146        g:tabwins_max_h_size
147 
148     => for dimensions >9 with :Tabwins*(), use a list arg, as above.
149 
150 
151   2) :V{I}x{J}  & :H{I}x{J}, :TabwinsVertical {J+}, :TabwinsVertical { [ J,J,... ] }
152      e.g. :V3x2 &     :H4x3, :TabwinsVertical  222, :TabwinsVertical   [ 12, 12, 12 ]
153                              :Tabwins          222, :Tabwins           [ 12, 12, 12 ]
154 
155     Two-dimensional tabs, w/ primary axis identifed by
156     V or H.
157 
158     E.g. V3x2 creates a tab of 3 vertical spaces, and inside
159     each vertical space, 2 horizontal windows. Whereas H4x3
160     means a tab of 4 rows, each w/ 3 windows.
161 
162     The window structures in group 1) are all in
163     this group, but here with names like V5x1 rather
164     than V5.
165 
166     Loading tabwins.vim creates :VIxJ & :HIxJ commands where
167     the set of {I}x{J} is the permutations of:
168        g:tabwins_max_v_size
169        g:tabwins_max_h_size
170 
171     The :TabwinsVertical/Horizontal form accepts arbitrary dimension
172     specification, unconstrained by configuration vars, other than
173     window-per-tab limits of the Vim instance.
174 
175     => for dimensions >9 with :Tabwins*(), use a list arg , as above.
176 
177 
178 
179   3) :TabwinsVertical/Horizontal {J+}, :TabwinsVertical/Horizontal  { [ I, J,... ] }
180      e.g. :TabwinsVertical 273, :TabwinsVertical [ 12, 7, 16, 2 ]
181           :Tabwins         273, :Tabwins         [ 12, 7, 16, 2 ]
182 
183     Tabs w/ primary axis identifed as Vertical or Horizontal in the
184     command name.  Each integer specifies the number of windows
185     inside each successive vertical or horizontal space.
186 
187     E.g. V273 creates a tab w/ 3 vertical spaces, the left having
188     2 horizontal windows, the middle 7, and the right 3.
189 
190     => for dimensions >9 with :Tabwins*(), use a list arg , as above.
191 
192 
193 
194   POPULATED WINDOWS
195 
196   These can be both symmetric and asymmetric, for both vertical and
197   horizontal priority tabs.  The developer specifies the content that
198   fills each window buffer.  See the examples in the Tabwins menu,
199   as well as documentation in TabwinsAddingNewBuilders.
200 
201 
202 
203   OPERATING SYSTEMS
204 
205   Core code has run successfully for gvim on Linux, Sun Solaris, HPUX,
206   iMac OS X, MSWIN, & cygwin.
207 
208   On MSWIN, demos of populated tabs may fail to load some buffer
209   content due to MSWIN file system syntax conventions.

213 Section 3: Vertical & Horizontal                    TabwinsVandH
214 ====================================================================
215 
216 Windows are built one axis at a time. The first axis built
217 is the primary axis, and the other axis is the secondary axis.
218        ~~~~~~~~~~~~                            ~~~~~~~~~~~~~~
219 If the tab starts by building the vertical axis, then the
220 vertical axis is the primary axis, and the horizontal axis
221 is the secondary axis. And vice-versa.
222 
223 What's the difference between Horizontal and Vertical Tabs ?
224 
225   Structurally, V2x3 is the same as H3x2.  The only difference
226   is how Vim numbers the tabs, based on the axis that is the
227   primary axis.
228 
229   IF the primary axis is vertical, then numbering starts in
230   col 1 and goes down the windows in col 1, then to the top
231   of col 2, then down col 2, and so on.
232 
233   IF the primary axis is horizontal, numbering starts in
234   row 1 and goes left to right, then to row 2, left to right,
235   and so on.
236 
237   Most of the asymmetric tab builders defined in the plugin
238   use a vertical primary axis.  Developers can define any
239   mix of primary axis in their own tab builders -- See
240   TabwinsAddingNewBuilders

243 Section 4: Tabwins Menu                                TabwinsMenu
244 ====================================================================
245 
246 In the Tabwins menu delivered with the plugin, the
247 extent that populated tabs fill their buffers is dependent
248 on the shell vars available to the Vim session, and files/dirs
249 present in the local environment.
250 
251 The menu items reference some of these shell vars.
252 
253 The developer is encouraged to customize tab builders to match
254 local usage.
255 
256 Update or replace func
257   Tabwins_menu_build()
258 in tabwins.vim to include any set of tab builders.

262 Section 5: Configuration                      TabwinsConfiguration
263 ====================================================================
264 
265 The below globals are defined in tabwins.vim and are overridable.
266 
267 To prevent 'Tabwins' menu addition when tabwins loads:
268   let g:load_tabwins_menu_is_wanted       = 'N'
269 
270 To change the 'Tabwins' menu number:
271   let g:tabwins_menu_number = {menu_number}
272 
273 To change the default top-line of netrw windows:
274   let g:tabwins_netrw_line_number_default = {N}
275 Calls to Fill_tab() can override this on a per-tab basis.
276 
277 To change the default netrw liststyle:
278   let g:tabwins_netrw_liststyle_default   = {N}
279 Calls to Fill_tab() can override this on a per-tab basis.
280 
281 To change V/H max dimensions:
282   let g:tabwins_max_v_size                = {N}
283   let g:tabwins_max_h_size                = {N}

287 Section 6: Adding New Builders            TabwinsAddingNewBuilders
288 ====================================================================
289 
290 1) ADDING SYMMETRIC TAB COMMANDS  (Empty Buffers)
291 
292 Bump either/both values of:
293   g:tabwins_max_v_size
294   g:tabwins_max_h_size
295 above the values assigned in tabwins.vim. Loading tabwins.vim will
296 build commands for all the permutations of the size vars.
297 
298 Of course, the set of symmetric tab builder commands can be reduced by
299 updating the same globals.
300 
301 
302 
303 2) ADDING ASYMMETRIC TAB COMMANDS  (Empty Buffers)
304 
305 Copy and update any of the
306   Create_tab_v{J...}()    
307   Create_tab_h{J...}()
308 functions.
309 
310 See the comments in the asymmetric command building code.
311 
312 
313 
314 3) ADDING COMMANDS FOR TABS OF POPULATED BUFFERS
315 
316 Update, clone, or replace any of the
317   Open_tab_*()
318 functions in tabwins.vim to create populated window structures.
319 
320 Example in tabwins.vim: 
321 
322       " ---------------------------------------------------------------
323       function! Open_tab_home_dir()
324       " ---------------------------------------------------------------
325         "1ST, Create the windowed-tab
326         :V232
327         "VERTICAL ASYMMETRIC TAB.  3 cols, from left to right with
328         "2, 3, and then 2 windows successively.
329 
330         let l:home_parent_dirpath = substitute(finddir($HOME), '\/\w\+$', '', 'g')
331 
332         "2ND, Fill buffers in the new tab
333         call Fill_tab({
334         \ 'line_number_at_window_top' : 1,
335         \ 'window_fill_specs' : [
336         \   $HOME,
337         \   'Explore ' . l:home_parent_dirpath,
338         \
339         \   'edit! ~/.bashrc',
340         \   '~/.gitignore',
341         \   'enew!',
342         \
343         \   '~/.vimrc',
344         \   '~/.vim',
345         \ ]
346         \})
347         "Fill_tab() fills windows in the order of elements
348         "in 'window_fill_specs' You want to spec something for each
349         "of the windows created by :V232 above,
350         "either commands, filepaths, or dirpaths.
351         "Specs can include shell or Vim vars, and if files or dirs
352         "the spec can include a Vim command or use the default :edit or
353         ":Explore call in Fill_tab()
354 
355         "3d, optional, apply Vim resize and/or other commands to the windows/buffers.
356         2wincmd w
357         resize 30
358 
359         1wincmd w
360         "And end back in window 1
361       endfunction
362 
363       NOTE: Since strings that specify commands are submitted to :execute,
364       it should be possible to embed multiple commands in one of these
365                                ~~~~~~~~~~~~~~~~~~~~~~~
366       strings, e.g. the resize commands below could be included via the
367       Vim \| command concatenator.  This is not yet validated -- the
368       developer is encouraged to try this. Notes also that if resizes of
369       multiple windows are intended, the order of the resizes can affect
370       the result, so it may sometimes still be preferable to resize
371       after calling Fill_tab()
372 
373 
374 Every window-populating tab-builder calls Fill_tab(), with signature: 
375 
376       'explore_cmd_name'          : 'Explore',
377       'window_fill_specs'         : [],
378       'netrw_liststyle'           : g:tabwins_netrw_liststyle_default,
379       'line_number_at_window_top' : g:tabwins_netrw_line_number_default,
380       'first_window_number'       : 1,
381       'ending_window_number'      : 1,
382 
383 See the Open_tab_*() functions for examples of overriding Fill_tab()
384 defaults.
385 
386 See the NOTE: comments in tabwins.vim for more information. Note particularly
387 the directives about window deletion order in calls to Close_windows().
