 35 Section 1: Introduction                         TabwinsIntroduction
 36 ====================================================================
 37 
 38 See the screen shots at
 39   https://github.com/cesheridan/graphics/tree/master/tabwins
 40 
 41 
 42 tabwins.vim creates and enables definition of Vim commands that
 43 BUILD TABS with:
 44 
 45   1) SYMMETRIC  window structures of EMPTY BUFFERS 
 46 
 47      e.g. for 4 columns, each with 2 windows --
 48        :TabwinsVertical 2222  OR
 49        :V4x2
 50 
 51 
 52      e.g. for 2 columns, each with 12 windows --
 53         :TabwinsVertical [ 12, 12 ]  OR
 54         :V2x12
 55 
 56 
 57 
 58   2) ASYMMETRIC window structures of EMPTY BUFFERS
 59 
 60      e.g. for 4 columns, of 2, 1, 1, & 3 windows successively --
 61        :TabwinsVertical 2113
 62 
 63      e.g. for 4 columns, of 18, 5, 26, & 3 windows successively --
 64        :TabwinsVertical  [ 18, 5, 26, 3 ]
 65 
 66        (sizes >9 must use list format)
 67 
 68 
 69   3) EITHER of the above structures, with BUFFERS POPULATED
 70      by files, dirs, or results of running :execute on
 71      developer-specified strings.
 72 
 73      e.g. :Open_tab_unix_filesystem_1 for an asymmetric V234 tab
 74      populated with typical unix dirs.
 75 
 76 Loading tabwins.vim creates Vim commands that have name formats
 77 as described in section TabwinsUsage of tabwins.txt.
 78 
 79 Default settings load Vim menu Tabwins, for access to selected
 80                       ~~~~~~~~~~~~~~~~
 81 tab-builder commands.  Those and additional tab builders are
 82 available via Vim commands defined in tabwins.vim. The menu has
 83 only a small portion of the tab builders defined in tabwins.vim,
 84 and the developer is encouraged to add tab builder commands as needed.
 85 
 86 
 87 By enabling customized tab creation ~during~ Vim sessions,
 88 tabwins.vim gives every developer immediate access to the Vim
 89 window structures and content that the develoer most commonly
 90 uses.  The develoer avoids having to reload backed up Vim
 91 sessions,  avoids losing the current configuration, and
 92 avoids the  loss of focus caused by these discontinuities.
 93 
 94 Easy access to the power of the Vim GUI.
 95 
 96 This plugin is structured for deployment in a pathogen-managed
 97 bundle directory.
 98 
 99 
100 See the rest of tabwins.txt for more information.

104 Section 2: Usage                                       TabwinsUsage
105 ====================================================================
106 
107   See TabwinsVandH for background about Vertical/Horizontal
108   and primary/secondary axes.
109 
110   Loading tabwins.vim creates Vim commands that have name formats
111   as described below.
112 
113 
114   WINDOWS OF EMPTY BUFFERS, SYMMETRIC & ASYMMETRIC
115 
116   1) :V{N},  :H{N}, :TabwinsVertical {N}, :TabwinsVertical { [ I ] }
117      e.g. :V3, :H4, :TabwinsVertical   4, :TabwinsVertical [ 12 ]
118 
119     One-dimensional tabs, w/ primary axis identifed by V or H, and
120     w/ N windows.
121 
122     E.g. :V3 creates a tab of 3 vertical windows, and :H4 creates
123     a tab of 4 horizontal windows.
124 
125     Loading tabwins.vim creates :HN & :VN commands where N is from
126     1 to the value of the relevant axis size in:
127        g:tabwins_max_v_size
128        g:tabwins_max_h_size
129 
130     For dimensions >9, the list form of :TabwinsVertical is required.
131 
132 
133 
134   2) :V{I}x{J}  & :H{I}x{J}, :TabwinsVertical {J+}, :TabwinsVertical { [ J,J,... ] }
135      e.g. :V3x2 &     :H4x3, :TabwinsVertical  222, :TabwinsVertical   [ 12, 12, 12 ]
136 
137     Two-dimensional tabs, w/ primary axis identifed by
138     V or H.
139 
140     E.g. V3x2 creates a tab of 3 vertical spaces, and inside
141     each vertical space, 2 horizontal windows. Whereas H4x3
142     means a tab of 4 rows, each w/ 3 windows.
143 
144     The window structures in group 1) are all in
145     this group, but here with names like V5x1 rather
146     than the simpler V5.
147 
148     Loading tabwins.vim creates :VIxJ & :HIxJ commands where
149     the set of {I}x{J} is the permutations of:
150        g:tabwins_max_v_size
151        g:tabwins_max_h_size
152 
153     The :TabwinsVertical/Horizontal form accepts arbitrary dimension
154     specification, unconstrained by configuration vars, other than
155     window-per-tab limits of the Vim instance.
156 
157 
158 
159   3) :TabwinsVertical/Horizontal {J+}, :TabwinsVertical/Horizontal  { [ I, J,... ] }
160      e.g. :TabwinsVertical 273, :TabwinsVertical [ 12, 7, 16, 2 ]
161 
162     Tabs w/ primary axis identifed as Vertical or Horizontal in the
163     command name.  Each integer specifies the number of windows
164     inside each successive vertical or horizontal space.
165 
166     E.g. V23 creates a tab w/ 2 vertical spaces, the left having
167     2 horizontal windows, and the right having 3 horizontal windows.
168     Whereas H14 means two rows, the top w/ 1 window and the
169     lower row w/ 4 windows.
170 
171     Loading tabwins.vim creates the V{J...} & H{J...} commands
172     defined by functions
173       Create_tab_v*() and
174       Create_tab_h*().
175 
176     NOTE: tabwins.vim Release 1.8.0 
177     This release adds :TabwinsVertical and :TabwinsHorizontal,
178     enabling arbitrary dimensions for BOTH symmetrical and
179     asymmetrical window structures, within display limits of
180     the Vim instance.  Consequently, the hard-coded asymmetrical
181     commands of release 1.7.0, e.g :V113, will likely be removed
182     in a future release.
183 
184 
185 
186   POPULATED WINDOWS
187 
188   These can be both symmetric and asymmetric, for both vertical and
189   horizontal priority tabs.  The developer specifies the content that
190   fills each window buffer.  See the examples in the Tabwins menu,
191   as well as documentation in TabwinsAddingNewBuilders.
192 
193 
194   OPERATING SYSTEMS
195 
196   Core code has run successfully for gvim on Linux, Sun Solaris, HPUX,
197   iMac OS X, MSWIN, & cygwin.
198 
199   On MSWIN, demos of populated tabs may fail to load some buffer
200   content due to MSWIN file system syntax conventions.

204 Section 3: Vertical & Horizontal                    TabwinsVandH
205 ====================================================================
206 
207 Windows are built one axis at a time. The first axis built
208 is the primary axis, and the other axis is the secondary axis.
209        ~~~~~~~~~~~~                            ~~~~~~~~~~~~~~
210 If the tab starts by building the vertical axis, then the
211 vertical axis is the primary axis, and the horizontal axis
212 is the secondary axis. And vice-versa.
213 
214 What's the difference between Horizontal and Vertical Tabs ?
215 
216   Structurally, V2x3 is the same as H3x2.  The only difference
217   is how Vim numbers the tabs, based on the axis that is the
218   primary axis.
219 
220   IF the primary axis is vertical, then numbering starts in
221   col 1 and goes down the windows in col 1, then to the top
222   of col 2, then down col 2, and so on.
223 
224   IF the primary axis is horizontal, numbering starts in
225   row 1 and goes left to right, then to row 2, left to right,
226   and so on.
227 
228   Most of the asymmetric tab builders defined in the plugin
229   use a vertical primary axis.  Developers can define any
230   mix of primary axis in their own tab builders -- See
231   TabwinsAddingNewBuilders

234 Section 4: Tabwins Menu                                TabwinsMenu
235 ====================================================================
236 
237 In the Tabwins menu delivered with the plugin, the
238 extent that populated tabs fill their buffers is dependent
239 on the shell vars available to the Vim session, and files/dirs
240 present in the local environment.
241 
242 The menu items reference some of these shell vars.
243 
244 The developer is encouraged to customize tab builders to match
245 local usage.
246 
247 Update or replace func
248   Tabwins_menu_build()
249 in tabwins.vim to include any set of tab builders.

253 Section 5: Configuration                      TabwinsConfiguration
254 ====================================================================
255 
256 The below globals are defined in tabwins.vim and are overridable.
257 
258 To prevent 'Tabwins' menu addition when tabwins loads:
259   let g:load_tabwins_menu_is_wanted       = 'N'
260 
261 To change the 'Tabwins' menu number:
262   let g:tabwins_menu_number = {menu_number}
263 
264 To change the default top-line of netrw windows:
265   let g:tabwins_netrw_line_number_default = {N}
266 Calls to Fill_tab() can override this on a per-tab basis.
267 
268 To change the default netrw liststyle:
269   let g:tabwins_netrw_liststyle_default   = {N}
270 Calls to Fill_tab() can override this on a per-tab basis.
271 
272 To change V/H max dimensions:
273   let g:tabwins_max_v_size                = {N}
274   let g:tabwins_max_h_size                = {N}

278 Section 6: Adding New Builders            TabwinsAddingNewBuilders
279 ====================================================================
280 
281 1) ADDING SYMMETRIC TAB COMMANDS  (Empty Buffers)
282 
283 Bump either/both values of:
284   g:tabwins_max_v_size
285   g:tabwins_max_h_size
286 above the values assigned in tabwins.vim. Loading tabwins.vim will
287 build commands for all the permutations of the size vars.
288 
289 Of course, the set of symmetric tab builder commands can be reduced by
290 updating the same globals.
291 
292 
293 
294 2) ADDING ASYMMETRIC TAB COMMANDS  (Empty Buffers)
295 
296 Copy and update any of the
297   Create_tab_v{J...}()    
298   Create_tab_h{J...}()
299 functions.
300 
301 See the comments in the asymmetric command building code.
302 
303 
304 
305 3) ADDING COMMANDS FOR TABS OF POPULATED BUFFERS
306 
307 Update, clone, or replace any of the
308   Open_tab_*()
309 functions in tabwins.vim to create populated window structures.
310 
311 Example in tabwins.vim: 
312 
313       " ---------------------------------------------------------------
314       function! Open_tab_home_dir()
315       " ---------------------------------------------------------------
316         "1ST, Create the windowed-tab
317         :V232
318         "VERTICAL ASYMMETRIC TAB.  3 cols, from left to right with
319         "2, 3, and then 2 windows successively.
320 
321         let l:home_parent_dirpath = substitute(finddir($HOME), '\/\w\+$', '', 'g')
322 
323         "2ND, Fill buffers in the new tab
324         call Fill_tab({
325         \ 'line_number_at_window_top' : 1,
326         \ 'window_fill_specs' : [
327         \   $HOME,
328         \   'Explore ' . l:home_parent_dirpath,
329         \
330         \   'edit! ~/.bashrc',
331         \   '~/.gitignore',
332         \   'enew!',
333         \
334         \   '~/.vimrc',
335         \   '~/.vim',
336         \ ]
337         \})
338         "Fill_tab() fills windows in the order of elements
339         "in 'window_fill_specs' You want to spec something for each
340         "of the windows created by :V232 above,
341         "either commands, filepaths, or dirpaths.
342         "Specs can include shell or Vim vars, and if files or dirs
343         "the spec can include a Vim command or use the default :edit or
344         ":Explore call in Fill_tab()
345 
346         "3d, optional, apply Vim resize and/or other commands to the windows/buffers.
347         2wincmd w
348         resize 30
349 
350         1wincmd w
351         "And end back in window 1
352       endfunction
353 
354       NOTE: Since strings that specify commands are submitted to :execute,
355       it should be possible to embed multiple commands in one of these
356       strings, e.g. the resize commands below could be included via the
357       Vim \| command concatenator.  This is not yet validated -- the
358       developer is encouraged to try this.
359 
360 
361 Every window-populating tab-builder calls Fill_tab(), with signature: 
362 
363       'explore_cmd_name'          : 'Explore',
364       'window_fill_specs'         : [],
365       'netrw_liststyle'           : g:tabwins_netrw_liststyle_default,
366       'line_number_at_window_top' : g:tabwins_netrw_line_number_default,
367       'first_window_number'       : 1,
368       'ending_window_number'      : 1,
369 
370 See the Open_tab_*() functions for examples of overriding Fill_tab()
371 defaults.
372 
373 See the NOTE: comments in tabwins.vim for more information. Note particularly
374 the directives about window deletion order in calls to Close_windows().
