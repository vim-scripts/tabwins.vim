 34 Section 1: Introduction                         TabwinsIntroduction
 35 ====================================================================
 36 
 37 See the screen shots at
 38   https://github.com/cesheridan/graphics/tree/master/tabwins
 39 
 40 1 command for immediate access to Vim customized window structures
 41 and content ~during~ Vim sessions.
 42 
 43 The developer
 44   - Avoids hit-or-miss, time-consuming, manual, repetitious
 45     runs of :split/:vsplit, and thought-sinks about how
 46     to produce an intended window structure
 47 
 48   - Avoids having to reload backed-up Vim sessions that
 49     have the required window structures, thus losing the
 50     current configuration
 51 
 52   - Avoids loss of focus from these discontinuities
 53 
 54 Easy access to the power of the Vim GUI.
 55 
 56 
 57 COMMAND FORMS
 58 
 59   :Tabwins {window_dimensions}; {fill_content,...};  {tab_options, ...}
 60   :{symmetric_command}          {fill_content,...};  {tab_options, ...}
 61 
 62     => semicolons separate arg groups, commas separate inside groups.
 63 
 64 EXAMPLES
 65 
 66   " For a new tab of 4 columns of 1, 2, 1, & 1 windows successively
 67   " left to right, with buffers empty --
 68   :Tabwins 1211
 69 
 70   :Tabwins 1211;  $HOME, '~/.bashrc', '~/.vimrc', '/', '/usr/bin'
 71   " Populate the buffers
 72   " => 'fill_content' specs of files are automatically opened via :edit,
 73   "    while specs of dirs are opened via :Explore
 74   " (Specify an '' empty string for windows intended to be empty.)
 75 
 76   :Tabwins 11111; $HOME, '~/.bashrc', '~/.vimrc', '/', '/usr/bin'
 77   " Same buffer content, in different window structure, of 5 cols,
 78   " one window in each.
 79 
 80   :V5x1           $HOME, '~/.bashrc', '~/.vimrc', '/', '/usr/bin'
 81   " Structure & content as above -- since the structure is
 82   " SYMMETRIC, i.e. lengths of secondary axis dimensions are all
 83   " equal, this alternative is available to reduce keystrokes.
 84 
 85   :V5             $HOME, '~/.bashrc', '~/.vimrc', '/', '/usr/bin'
 86   " Same result again: since the secondary axis has only 1
 87   " window for its dimensions, V5 is available as an even
 88   " shorter alias.
 89 
 90   :Tabwins 1112;  $HOME, '~/.bashrc', '~/.vimrc', '/', 'Explore /usr/bin | resize 20'
 91   " Same content, different structure, which includes :resize of the last window,
 92   " via Vim '|' cmd concat.  If a file or dir is followed with concated commands,
 93   " the caller needs to explicitly add the command that opens the file or dir,
 94   " i.e. 'Explore' in this example.
 95

 98

100   an integer.  To enable dimensions >9, use list syntax:
101

104   " {fill_content} can be added as with all tabwins.vim commands.
105 
106   :Tabwins [ 18, 23 ]
107 
108   :Tabwins [ 14, 14 ]
109   " Alternately, :V2x14
110 
111   :Tabwins [ 35, 30, 40, 40 ]
112   " For large monitors ...

128 :Tabwins, :TabwinsVertical, & :TabwinsHorizontal
129 
130   :Tabwins is an alias for :TabwinsVertical, which gives
131   priority to the vertical axis.
132 
133   :TabwinsHorizontal gives priority to the horizontal axis.

136   & primary/secondary axes.

166 Section 2: Usage                                       TabwinsUsage
167 ====================================================================
168 
169 NOTE: tabwins.vim v2.1.0 enables Vim command-line spec of buffer 
170       fill and options.  The below text is still correct, but does 
171       not cover the v2.1.0 enhancements, which will be described in 
172       a later release.  The source code is open ... ;-)
173 
174   Loading tabwins.vim creates Vim commands that have name formats
175   as described below.
176 
177   See TabwinsVandH for background about Vertical/Horizontal
178   and primary/secondary axes.
179 
180 
181   DEFAULT AXIS :Tabwins 
182 
183   :Tabwins is an alias for :TabwinsVertical, meaning
184   that the Vertical axis is the default primary axis.
185 
186 
187   WINDOWS OF EMPTY BUFFERS, SYMMETRIC & ASYMMETRIC
188 
189   1) :V{N},  :H{N}, :TabwinsVertical {N}, :TabwinsVertical { [ I ] }
190      e.g. :V3, :H4, :TabwinsVertical   4, :TabwinsVertical [ 12 ]
191                     :Tabwins           4, :Tabwins [ 12 ]
192 
193     One-dimensional tabs, w/ primary axis identifed by V or H, and
194     w/ N windows.
195 
196     E.g. :V3 creates a tab of 3 vertical windows, and :H4 creates
197     a tab of 4 horizontal windows.
198 
199     Loading tabwins.vim creates :HN & :VN commands where N is from
200     1 to the value of the relevant axis size in:
201        g:tabwins_max_v_size
202        g:tabwins_max_h_size
203 
204     => for dimensions >9 with :Tabwins*(), use a list arg, as above.
205 
206 
207   2) :V{I}x{J}  & :H{I}x{J}, :TabwinsVertical {J+}, :TabwinsVertical { [ J,J,... ] }
208      e.g. :V3x2 &     :H4x3, :TabwinsVertical  222, :TabwinsVertical   [ 12, 12, 12 ]
209                              :Tabwins          222, :Tabwins           [ 12, 12, 12 ]
210 
211     Two-dimensional tabs, w/ primary axis identifed by
212     V or H.
213 
214     E.g. V3x2 creates a tab of 3 vertical spaces, and inside
215     each vertical space, 2 horizontal windows. Whereas H4x3
216     means a tab of 4 rows, each w/ 3 windows.
217 
218     The window structures in group 1) are all in
219     this group, but here with names like V5x1 rather
220     than V5.
221 
222     Loading tabwins.vim creates :VIxJ & :HIxJ commands where
223     the set of {I}x{J} is the permutations of:
224        g:tabwins_max_v_size
225        g:tabwins_max_h_size
226 
227     The :TabwinsVertical/Horizontal form accepts arbitrary dimension
228     specification, unconstrained by configuration vars, other than
229     window-per-tab limits of the Vim instance.
230 
231     => for dimensions >9 with :Tabwins*(), use a list arg , as above.
232 
233 
234   3) :TabwinsVertical/Horizontal {J+}, :TabwinsVertical/Horizontal  { [ I, J,... ] }
235      e.g. :TabwinsVertical 273, :TabwinsVertical [ 12, 7, 16, 2 ]
236           :Tabwins         273, :Tabwins         [ 12, 7, 16, 2 ]
237 
238     Tabs w/ primary axis identifed as Vertical or Horizontal in the
239     command name.  Each integer specifies the number of windows
240     inside each successive vertical or horizontal space.
241 
242     E.g. V273 creates a tab w/ 3 vertical spaces, the left having
243     2 horizontal windows, the middle 7, and the right 3.
244 
245     => for dimensions >9 with :Tabwins*(), use a list arg , as above.
246

248   POPULATED WINDOWS
249 
250   These can be both symmetric and asymmetric, for both vertical and
251   horizontal priority tabs.  The developer specifies the content that
252   fills each window buffer.  See the examples in the Tabwins menu,
253   as well as documentation in TabwinsAddingNewBuilders.

265 ====================================================================
266 Section 3: Vertical & Horizontal                    TabwinsVandH
267 ====================================================================
268 
269 Windows are built one axis at a time. The first axis built
270 is the primary axis, and the other axis is the secondary axis.
271        ~~~~~~~~~~~~                            ~~~~~~~~~~~~~~
272 If the tab starts by building the vertical axis, then the
273 vertical axis is the primary axis, and the horizontal axis
274 is the secondary axis. And vice-versa.
275 
276 What's the difference between Horizontal and Vertical Tabs ?
277 
278   Structurally, V2x3 is the same as H3x2.  The only difference
279   is how Vim numbers the tabs, based on the axis that is the
280   primary axis.
281 
282   IF the primary axis is vertical, then numbering starts in
283   col 1 and goes down the windows in col 1, then to the top
284   of col 2, then down col 2, and so on.
285 
286   IF the primary axis is horizontal, numbering starts in
287   row 1 and goes left to right, then to row 2, left to right,
288   and so on.
289 
290   Most of the asymmetric tab builders defined in the plugin
291   use a vertical primary axis.  Developers can define any
292   mix of primary axis in their own tab builders -- See
293   TabwinsAddingNewBuilders

296 Section 4: Tabwins Menu                                TabwinsMenu
297 ====================================================================
298 
299 In the Tabwins menu delivered with the plugin, the
300 extent that populated tabs fill their buffers is dependent
301 on the shell vars available to the Vim session, and files/dirs
302 present in the local environment.
303 
304 The menu items reference some of these shell vars.
305 
306 The developer is encouraged to customize tab builders to match
307 local usage.
308 
309 Update or replace func
310   Tabwins_menu_build()
311 in tabwins.vim to include any set of tab builders.

315 Section 5: Configuration                      TabwinsConfiguration
316 ====================================================================
317 
318 The below globals are defined in tabwins.vim and are overridable.
319 
320 To prevent 'Tabwins' menu addition when tabwins loads:
321   let g:load_tabwins_menu_is_wanted       = 'N'
322 
323 To change the 'Tabwins' menu number:
324   let g:tabwins_menu_number = {menu_number}
325 
326 To change the default top-line of netrw windows:
327   let g:tabwins_netrw_line_number_default = {N}
328 Calls to Fill_tab() can override this on a per-tab basis.
329 
330 To change the default netrw liststyle:
331   let g:tabwins_netrw_liststyle_default   = {N}
332 Calls to Fill_tab() can override this on a per-tab basis.
333 
334 To change V/H max dimensions:
335   let g:tabwins_max_v_size                = {N}
336   let g:tabwins_max_h_size                = {N}

340 Section 6: Adding New Builders            TabwinsAddingNewBuilders
341 ====================================================================
342 
343 NOTE: tabwins.vim v2.1.0 enables Vim command-line spec of buffer 
344       fill and options.  The below text is still correct, but does 
345       not cover the v2.1.0 enhancements, which will be described in 
346       a later release.  The source code is open ... ;-)
347 
348       One change to the below is that in v2.1.1 'fill_content' 
349       is the new identifier for the previous 'window_fill_specs'
350 
351 
352 1) ADDING SYMMETRIC TAB COMMANDS  (Empty Buffers)
353 
354 Bump either/both values of:
355   g:tabwins_max_v_size
356   g:tabwins_max_h_size
357 above the values assigned in tabwins.vim. Loading tabwins.vim will
358 build commands for all the permutations of the size vars.
359 
360 Of course, the set of symmetric tab builder commands can be reduced by
361 updating the same globals.
362 
363 
364 
365 2) ADDING ASYMMETRIC TAB COMMANDS  (Empty Buffers)
366 
367 Copy and update any of the
368   Create_tab_v{J...}()    
369   Create_tab_h{J...}()
370 functions.
371 
372 See the comments in the asymmetric command building code.
373 
374 
375 
376 3) ADDING COMMANDS FOR TABS OF POPULATED BUFFERS
377 
378 Update, clone, or replace any of the
379   Open_tab_*()
380 functions in tabwins.vim to create populated window structures.
381 
382 Example in tabwins.vim: 
383 
384       " ---------------------------------------------------------------
385       function! Open_tab_home_dir()
386       " ---------------------------------------------------------------
387         " 1ST, Create the windowed-tab
388         :Tabwins 232
389         " VERTICAL ASYMMETRIC TAB.  3 cols, from left to right with 2, 3,
390         " and 2 windows successively left to right.
391 
392         let l:home_parent_dirpath = substitute(finddir($HOME), '\/\w\+$', '', 'g')
393 
394         "2ND, Fill buffers in the new tab
395         "v2.1.1: 'fill_content' was previously 'window_fill_specs'
396         call Fill_tab({
397         \ 'line_number_at_window_top' : 1,
398         \ 'fill_content' : [
399         \   $HOME,
400         \   'Explore ' . l:home_parent_dirpath,
401         \
402         \   'edit! ~/.bashrc',
403         \   '~/.gitignore',
404         \   'enew!',
405         \
406         \   '~/.vimrc',
407         \   '~/.vim',
408         \ ]
409         \})
410 
411         " Fill_tab() fills windows in the order of elements
412         " in 'window_fill_specs' You want to spec something for each
413         " of the windows created by :Tabwins 232 above,
414         " either commands, filepaths, or dirpaths.
415         " Specs can include shell or vim vars, and if files or dirs
416         " the spec can include a vim command or use the default :edit or
417         " :Explore call in Fill_tab()
418 
419         "3d, optional, apply Vim resize and/or other commands to the windows/buffers.
420         2wincmd w
421         resize 30
422 
423         1wincmd w
424         "And end back in window 1
425       endfunction
426 
427       NOTE: Since strings that specify commands are submitted to :execute,
428       it should be possible to embed multiple commands in one of these
429                                ~~~~~~~~~~~~~~~~~~~~~~~
430       strings, e.g. the resize commands above could be included via the
431       Vim \| command concatenator.  This is not yet validated -- the
432       developer is encouraged to try this. Notes also that if resizes of
433       multiple windows are intended, the order of the resizes can affect
434       the result, so it may sometimes still be preferable to resize
435       after calling Fill_tab()
436 
437 
438 Every window-populating tab-builder calls Fill_tab(), with signature: 
439 
440       'explore_cmd_name'          : 'Explore',
441       'fill_content'              : [],
442       'netrw_liststyle'           : g:tabwins_netrw_liststyle_default,
443       'line_number_at_window_top' : g:tabwins_netrw_line_number_default,
444       'first_window_number'       : 1,
445       'ending_window_number'      : 1,
446 
447 See the Open_tab_*() functions for examples of overriding Fill_tab()
448 defaults.
449 
450 See comments in tabwins.vim for more information.
