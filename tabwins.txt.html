 37 Section 1: Introduction                         TabwinsIntroduction
 38 ====================================================================
 39 
 40 Easy access to the power of the Vim GUI.
 41 
 42 See the screen shots at
 43   https://github.com/cesheridan/graphics/tree/master/tabwins
 44 
 45 
 46 1 command for immediate access to customized window structures
 47 and content ~during~ Vim sessions.
 48 
 49 The Developer
 50   - Avoids hit-or-miss, time-consuming, manual, repetitious
 51     runs of :split/:vsplit, and thought-sinks about how
 52     to produce an intended window structure
 53 
 54   - Avoids losing current configurations due to mid-session
 55     reloads of backed-up Vim sessions that have preferred
 56     window structures
 57 
 58   - Avoids loss of focus from these discontinuities
 59 
 60 
 61 FORMS --
 62 
 63   :{Tabwins cmd}   {window_dimensions}; {fill_content,...};  {tab_option, ...}
 64   :{symmetric cmd}                      {fill_content,...};  {tab_option, ...}
 65 
 66   Where
 67    :{Tabwins cmd}   is any of :Tabwins, :TabwinsVertical, :TabwinsHorizontal
 68    :{symmetric cmd} is any of the form :{V|H}IxJ or :{V|H}I
 69 
 70   Semicolons separate arg groups, commas separate inside groups.
 71 
 72   {window_dimensions} is the only mandatory arg, only in :{Tabwins cmd}
 73 
 74   SECTION TabwinsUsage IN tabwins.txt HAS MORE DETAIL.
 75 
 76 
 77 EXAMPLES --
 78 
 79   >> Empty Windows
 80 
 81   :Tabwins 1211
 82   " For a new tab of 4 columns of 1, 2, 1, & 1 windows successively
 83   " left to right, with buffers empty.
 84 
 85   :TabwinsHorizontal 4221
 86   " For a new tab of 4 rows, with 4,2,2, & 1 windows successively
 87   " top to bottom, with buffers empty.
 88 
 89 
 90   >> Populated Windows
 91 
 92   :Tabwins            1211; $HOME, '~/.bashrc', '~/.vimrc', '/', '/usr/bin'
 93   " Populate the buffers
 94   " => 'fill_content' specs of files are automatically opened via :edit,
 95   "    while specs of dirs are opened via :Explore
 96   " (Specify an '' empty string for windows intended to be empty.)
 97 
 98   :Tabwins           11111; $HOME, '~/.bashrc', '~/.vimrc', '/', '/usr/bin'
 99   " Same buffer content, in different window structure, of 5 cols,
100   " one window in each.
101 
102   :TabwinsHorizontal 11111; $HOME, '~/.bashrc', '~/.vimrc', '/', '/usr/bin'
103   " Flip axis priority of above to the horizontal
104 
105   :Tabwins             312; '~/.bashrc', '~/.vimrc', '~/.git', $HOME, '/', '/usr/bin'
106 
107 
108   >> Symmetric Window Structures
109 
110   :V5x8
111   :H5x8
112   :V8x14
113   :H8x14
114   "Empty windows
115 
116   :V5x1 $HOME, '~/.bashrc', '~/.vimrc', '/', '/usr/bin'
117   " Structure & content same as :Tabwins 11111 above
118 
119   :V5   $HOME, '~/.bashrc', '~/.vimrc', '/', '/usr/bin'
120   " Same result again: since the secondary axis has only 1
121   " window for its dimensions, V5 is available as an even
122   " shorter alias.
123 
124 
125   >> Dimensions >9
126 
127   :Tabwins [ 5, 8, 13, 21 ]
128   " For a 4-column tab with 5, 8, 13, 21 windows successively left to right.
129 
130   :Tabwins [ 14, 14 ]
131   " Alternately, :V2x14
132 
133   :Tabwins [ 40, 30, 40, 30 ]
134   " For large monitors ...
135 
136 
137   >> fill_specs with Multiple Commands
138 
139   :Tabwins 1112;  $HOME, '~/.bashrc', '~/.vimrc', '/', 'Explore /usr/bin | resize 20'
140   " Does a :resize of the last window, via Vim '|' command concat.  If a file
141   " or dir is followed with concated commands, the caller needs to explicitly
142   " add the command that opens the file or dir, i.e. 'Explore' in this example.
143 
144 
145   >> Tab Options
146 
147   :Tabwins 1211;  $HOME, '~/.bashrc', '~/.vimrc', '/', '/usr/bin'; 'ending_window_number' : 2
148   :Tabwins 1211;  $HOME, '~/.bashrc', '~/.vimrc', '/', '/usr/bin'; 'netrw_liststyle'      : 3
149 
150   :Tabwins 1211;  $HOME, '~/.bashrc', '~/.vimrc', '/', '/usr/bin'; 'netrw_liststyle'      : 3, 'ending_window_number' : 2
151   " Multiple tab_options in same command.
152 
153 TABWINS MENU --
154 
155   Default settings load Vim menu Tabwins, for access to selected
156                         ~~~~~~~~~~~~~~~~
157   tab-builder commands.  Those and additional tab builders are
158   available via Vim commands defined in tabwins.vim. The menu has
159   only a small portion of the tab builders defined in tabwins.vim.
160 
161 
162 DEPLOYMENT --
163 
164   tabwins.vim tabwins/ dir is structured for deployment in a
165   pathogen-managed bundle directory, e.g. ~/.vim/bundle/
166 
167 
168   In a ~/.vim dir not configured for pathogen, put
169     tabwins/plugin/* into ~/.vim/plugin/
170   and
171     tabwins/doc/*    into ~/.vim/doc/
172 
173 
174   Enable :h tabwins in a pathogen environment with
175     :Helptags
176   and in a non-pathogen environment with
177     :helptags

181 Section 2: Usage                                       TabwinsUsage
182 ====================================================================
183 
184 => See FORMS    in TabwinsIntroduction
185 => See EXAMPLES in TabwinsIntroduction
186 
187 
188 THREE TABWINS COMMANDS --
189 
190   These commands all take the same arguments.
191 
192   :Tabwins
193     Same as :TabwinsVertical
194 
195   :TabwinsVertical
196     Gives priority to the vertical axis.
197 
198   :TabwinsHorizontal 
199     Gives priority to the horizontal axis.
200 
201   See TabwinsVandH about primary/secondary axes and
202   vertical/horizontal priority.
203 
204   These commands accept arbitrary dimension specification,
205   unconstrained by configuration vars, other than
206   windows-per-tabsize limits of the Vim instance.
207 
208 
209 SYMMETRIC WINDOW STRUCTURES --
210 
211   In these, the lengths of the secondary axis dimensions are all
212   equal.
213 
214   Symmetric commands are available to reduce keystrokes.
215 
216   'H' indicates horizontal priority, 'V' vertical priority.
217 
218   Loading tabwins.vim creates :{V|H}IxJ commands where the
219   set of IxJ is the permutations of:
220     g:tabwins_max_v_size 
221     g:tabwins_max_h_size 
222 
223   The same global vars determine the set of :{V|H}J commands
224   also produced on tabwins.vim load, where N is from 1 to the
225   value of the relevant g:tabwins_max_{h|v}_size var above.
226 
227 
228 DIMENSIONS GREATER THAN 9 --
229 
230   Dimensions >9 are unparsable when {window_dimensions} is
231   an integer.  To enable dimensions >9, use list syntax as
232   shown in EXAMPLES of TabwinsIntroduction
233 
234   List syntax applies to any tab that has dimension >9.
235   All the dimensions of such a tab are specified in one list.
236 
237 
238 FILL SPECS WITH MULTIPLE COMMANDS --
239 
240   Multiple Vim commands can be specified in a fill_spec by
241   prepending commands with the Vim '|' concatenation operator.
242   See EXAMPLES in TabwinsIntroduction.
243 
244 
245 TAB OPTIONS --
246 
247   Available tab_options are as below, with defaults on
248   the right.
249 
250   These apply to the entire tab and can be combined when
251   separated by commas.
252 
253      'explore_cmd_name'          : 'Explore',
254      'netrw_liststyle'           : g:tabwins_netrw_liststyle_default,
255      'line_number_at_window_top' : g:tabwins_netrw_line_number_default,
256 
257      => Use a fill_spec for window-specific override of the above.
258 
259      'first_window_number'       : 1,
260      'ending_window_number'      : 1
261 
262 
263   See EXAMPLES in TabwinsIntroduction.
264 
265   Function Fill_tab() processes tab_options.
266 
267 
268 HELP --
269 
270   :TabwinsFindTabwins will load the tabwins .vim & .txt files
271   into a V2 window, if your Vim &runtimepath can see them.
272 
273   This command is included in the 'Tabwins' menu.

277 Section 3: Vertical & Horizontal                    TabwinsVandH
278 ====================================================================
279 
280 Windows are built one axis at a time. The first axis built
281 is the primary axis, and the other axis is the secondary axis.
282        ~~~~~~~~~~~~                            ~~~~~~~~~~~~~~
283 If the tab starts by building the vertical axis, then the
284 vertical axis is the primary axis, and the horizontal axis
285 is the secondary axis. And vice-versa.
286 
287 What's the difference between Horizontal and Vertical Tabs ?
288 
289   Structurally, V2x3 is the same as H3x2.  The only difference
290   is how Vim numbers the tabs, based on the axis that is the
291   primary axis.
292 
293   IF the primary axis is vertical, then numbering starts in
294   col 1 and goes down the windows in col 1, then to the top
295   of col 2, then down col 2, and so on.
296 
297   IF the primary axis is horizontal, numbering starts in
298   row 1 and goes left to right, then to row 2, left to right,
299   and so on.
300 
301   Most of the asymmetric tab builders defined in the plugin
302   use a vertical primary axis.  Developers can define any
303   mix of primary axis in their own tab builders -- See
304   TabwinsAddingNewBuilders

308 Section 4: 'Tabwins' Menu                               TabwinsMenu
309 ====================================================================
310 
311 In the 'Tabwins' menu built on plugin load, the extent that
312 populated tabs fill their buffers is dependent on the shell
313 vars available to the Vim session, and files/dirs present in
314 the local environment.
315 
316 The menu items reference some of these shell vars.
317 
318 The developer is encouraged to customize this menu for local usage.
319 
320 Update or replace func
321   function Tabwins_menu_build()
322 in tabwins.vim to do so.

326 Section 5: Configuration                      TabwinsConfiguration
327 ====================================================================
328 
329 The below globals are defined in tabwins.vim and are overridable.
330 
331 To limit tabwins.vim load to once per Vim session:
332   let g:tabwins_multiple_loads_are_permitted = 'N'
333 Default: 'Y'
334 
335 To prevent 'Tabwins' menu addition when tabwins loads:
336   let g:load_tabwins_menu_is_wanted          = 'N'
337 Default: 'Y'
338 
339 To change the 'Tabwins' menu number:
340   let g:tabwins_menu_number                  = {menu_number}
341 Default: 9998
342 
343 To change the default top-line of netrw windows:
344   let g:tabwins_netrw_line_number_default    = {N}
345 Default: g:tabwins_netrw_line_number_dirpath = 3
346 Calls to Fill_tab() can override this on a per-tab basis.
347 
348 To change the default netrw liststyle:
349   let g:tabwins_netrw_liststyle_default      = {N}
350 Default: g:tabwins_netrw_liststyle_default   = 1
351 Calls to Fill_tab() can override this on a per-tab basis.
352 if netrw var g:netrw_liststyle is not defined, it gets this value.
353 
354 To change Vertical/Horizontal max dimensions:
355   let g:tabwins_max_v_size                   = {N}
356   let g:tabwins_max_h_size                   = {N} 
357 Default: 15 for each

361 Section 6: Adding New Builders            TabwinsAddingNewBuilders
362 ====================================================================
363 
364 
365 ADDING SYMMETRIC TAB COMMANDS --
366 
367   Bump either/both values of:
368     g:tabwins_max_v_size
369     g:tabwins_max_h_size
370 
371   above the values assigned in tabwins.vim.
372 
373   Loading tabwins.vim will build commands for all the HxV and
374   VxH permutations.
375 
376   Of course, the set of symmetric tab builder commands can be reduced by
377   lowering the values of the same globals.
